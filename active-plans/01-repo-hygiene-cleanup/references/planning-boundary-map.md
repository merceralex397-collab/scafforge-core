# Planning Boundary Map

## Canonical Plan Surfaces

- `../../README.md`
- `../../WORK-JOURNAL.md`
- `../../FULL-REPORT.md`
- `../../docscleanup.md`
- all numbered plan folders

These files own the current implementation program, sequencing, and cross-plan instructions.

## Active Supporting Reference Surfaces

- `../../_source-material/repo-hygiene/`
- `../../_source-material/downstream-failures/`
- `../../_source-material/asset-pipeline/`
- `../../_source-material/validation/`
- `../../_source-material/autonomy/`

These remain active documentation inside `active-plans/`. They are supporting references, not throwaway staging material.

## Decision Rule

If a file contains package-owned implementation instructions, sequencing logic, verification criteria, or cross-plan conclusions, it belongs in the canonical plan surfaces.

If a file contains copied docs, raw notes, exploratory research, or evidence that still informs package work, it belongs in the active supporting reference surfaces.

If a source file becomes fully superseded, keep an explicit pointer to the replacement plan note instead of assuming it should leave the repo.
