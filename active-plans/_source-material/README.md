# Active Source Material

This folder contains the raw inputs that existed in `active-plans/` before the reorganization:

- draft plans
- copied vendor or ecosystem documentation
- research dumps
- failure notes from downstream repos
- exploratory validation/tooling notes

These files remain active documentation inside `active-plans/`, but they are not the canonical implementation plans anymore.

Use the numbered plan folders for active work. Use `_source-material/` only when you need to trace the origin of a planning decision or extract more detail during implementation.

Keep this material in-repo unless there is a later, explicit decision to relocate or retire a specific branch. If a source note becomes fully superseded, add a pointer to the replacement plan or report so provenance stays clear.
