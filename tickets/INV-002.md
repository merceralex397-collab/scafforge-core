# INV-002: Extend stage-gate and pivot-state enforcement to the new invariant contract

## Summary

Extend preflight and pivot-state enforcement so runtime adapter calls, stage transitions, and pivot writers all obey the same mutation-time invariants and ownership boundaries.

## Why

Write-time invariants only work if every entrypoint, including adapter calls and pivot writers, is forced through the same contract. Otherwise hidden writers just move to a different script.

## Package Boundary

- Update package-owned preflight, adapter, and pivot enforcement surfaces.
- Do not add package-root trackers to compensate for weak enforcement.

## Do This

1. Extend `stage-gate-enforcer.ts` and related checks so invalid mutation requests fail before write time.
2. Make pivot-state mutation respect the single-owner contract established by `PIVOT-001`.
3. Ensure package-side adapter calls cannot bypass the same invariant checks as direct runtime tool calls.
4. Cover the hidden writer paths identified in the RFC, not just the obvious ticket update entrypoints.

## Files To Update

- `skills/repo-scaffold-factory/assets/project-template/.opencode/plugins/stage-gate-enforcer.ts`
- `skills/scafforge-pivot/scripts/pivot_tracking.py`
- shared runtime adapter or helper surfaces used by package callers

## Verify

1. Confirm invalid lifecycle or mutation requests are blocked before persistence.
2. Confirm pivot-state writes only occur through the allowed owner path.
3. Confirm adapter-driven mutations hit the same invariant checks as direct runtime tool calls.
