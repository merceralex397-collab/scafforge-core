# MUT-002: Route package-side ticket mutations through canonical runtime tools

## Summary

Route remediation follow-up and pivot lineage ticket creation or reconciliation through the canonical runtime mutation path so package-side orchestration no longer acts as a shadow ticket writer.

## Why

Even with a shared adapter, the rewrite fails if package-side follow-up scripts still write ticket state directly. This ticket closes that mutation seam.

## Package Boundary

- Update package follow-up scripts and pivot lineage code.
- Route generated behavior changes through template-owned runtime tools; do not add package-root ticket state.

## Do This

1. Replace direct guarded ticket writes in remediation follow-up with calls through the shared runtime bridge or an equivalent runtime-owned contract.
2. Move pivot lineage mutation through the same path.
3. Preserve split-scope, source-ticket, and follow-up linkage semantics while changing the writer.
4. Remove or isolate any remaining direct package-side ticket graph mutation logic.

## Files To Update

- `skills/ticket-pack-builder/scripts/apply_remediation_follow_up.py`
- `skills/scafforge-pivot/scripts/apply_pivot_lineage.py`
- shared runtime adapter or helper surfaces used by both

## Verify

1. Confirm follow-up tickets are still created with the right linkage and source semantics.
2. Confirm pivot lineage actions still work after the writer changes.
3. Confirm there is no remaining shadow ticket writer in the surfaces covered by this ticket.
