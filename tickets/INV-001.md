# INV-001: Enforce graph and lifecycle invariants at write time in workflow.ts

## Summary

Move graph, lifecycle, and active-ticket convergence checks into the runtime mutation path so invalid state cannot be written and then discovered later by audit or resume.

## Why

Audit cannot be the first place invalid graph or lifecycle state is discovered. The runtime mutation path has to reject contradictions before they are persisted.

## Package Boundary

- Implement invariants in package-owned template runtime code and package tests.
- Do not work around invalid states by tracking extra package-root status.

## Do This

1. Add write-time checks for graph, split-scope, follow-up, and lifecycle invariants in `workflow.ts`.
2. Enforce `manifest.active_ticket` and workflow convergence inside the runtime mutation path.
3. Reject impossible stage, status, resolution, and verification combinations before they are written.
4. Add tests that prove invalid mutations fail early.

## Files To Update

- `skills/repo-scaffold-factory/assets/project-template/.opencode/lib/workflow.ts`
- runtime mutation entrypoints that call into `workflow.ts`
- package validation coverage for mutation invariants

## Verify

1. Confirm invalid graph and lifecycle combinations are rejected at mutation time.
2. Confirm valid mutations still persist successfully.
3. Confirm the runtime path enforces active-ticket convergence instead of leaving it to later audit.
