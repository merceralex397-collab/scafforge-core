# Devdocs Index

This folder now separates **active working artifacts** from **historical analysis** so old review material does not compete with current operating guidance.

## Active docs in `devdocs/`

Keep these files at the top level because they were explicitly requested as current project artifacts:

- `scafforge-implementation-plan.md` — active implementation reference
- `worklog.md` — active implementation record
- `finalizationplan.md` — active finalization roadmap

## Historical docs in `devdocs/history/`

The `history/` subfolder contains older analysis, review, and improvement documents kept for reference only.

Treat those files as archival background, not as current source-of-truth operating guidance.

## Reading order for agents

When you need current repo guidance, prefer:

1. root `README.md`
2. root `AGENTS.md`
3. relevant `skills/*/SKILL.md` files
4. the three active docs listed above

Open `devdocs/history/` only when you need historical rationale, prior audits, or older research.
