## Root Cause Analysis

### Root cause A — Auto-detected completion in repair runner

The repair cycle `2026-04-07T21:12:10Z` recorded both `opencode-team-bootstrap` and `agent-prompt-engineering` as completed via `auto-detected`. The `repair-follow-on-state.json` confirms these were listed under `required_follow_on_stages` but then closed out without actual skill execution.

This means every agent in `.opencode/agents/` is still on the contract from the prior scaffold generation pass. The 30+ commit package rewrite introduced new template content (stop conditions, advancement rules, contradiction resolution, `review-audit-bridge` wiring for all reviewer/QA agents, the expanded ticket-creator with `ticket_reconcile`) — but none of that reached the repo because the agent regeneration step was auto-detected as done.

### Root cause B — No verification gate between "completion recorded" and "content confirmed"

The `scafforge-repair` flow did not require that the resulting agent files actually match the current template before recording stage completion. Because completion was inferred rather than proved, no content diff was run, and the drift persisted silently.

### Root cause C — `ticket_reconcile` capability absent from ticket-creator

The ticket-creator agent predates the addition of `ticket_reconcile` to the template. This is a capability gap: WFLOW019 and WFLOW024 reconciliation paths require the ticket-creator to also own `ticket_reconcile`, but the current repo version only has `ticket_create`.

### Root cause D — `review-audit-bridge` not wired into reviewer/QA agents

All three reviewer-code, reviewer-security, and tester-qa agents are missing `review-audit-bridge` skill permission and its body references. This was added in the package rewrite but was never applied to the repo because the agent regeneration stage auto-detected completion.

### Root cause E — Reviewer-code has no compile-check capabilities

The reviewer-code agent cannot run `python -m py_compile`, `cargo check`, `tsc --noEmit`, or equivalent compile checks. It has no bash allowlist entries for these commands. This means a PR that introduces a syntax error can receive a reviewer-code PASS without any verification.
