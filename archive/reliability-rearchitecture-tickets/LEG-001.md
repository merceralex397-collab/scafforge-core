# LEG-001: Implement the legacy workflow contract migration stage and escalation rules

## Summary

Add the explicit legacy contract migration stage so managed repair can safely upgrade recent repos, bound any compatibility shim to package-internal use, and escalate very old repos instead of mutating them silently.

## Why

The rewrite cannot rely on indefinite backward-compatibility drift. Legacy repos need a real upgrade path or an explicit escalation path.

## Package Boundary

- Implement migration in package repair code and package provenance surfaces.
- Do not hide migration state in package-root generated-repo runtime files.

## Do This

1. Add a dedicated migration stage ahead of ordinary repair mutation.
2. Define which legacy repos are safe to auto-upgrade and which must fail closed.
3. Keep any compatibility shim package-internal and temporary.
4. Record escalation conditions for structurally unsafe or too-old repos.

## Files To Update

- `skills/scafforge-repair/scripts/run_managed_repair.py`
- migration helpers introduced for the new stage
- provenance or escalation surfaces touched by migration

## Verify

1. Confirm recent legacy repos can enter the new contract through the migration stage.
2. Confirm very old or structurally inconsistent repos escalate instead of mutating silently.
3. Confirm the compatibility shim does not leak into generated output behavior.
