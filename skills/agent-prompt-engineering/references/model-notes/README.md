# Model Notes

This directory contains package-side model-family prompting references used during scaffolding.

Treat these notes as read-only package guidance during project generation.

These notes are not the canonical source for provider routing, exact model IDs, or fallback policy.

Verify exact model IDs and current provider routes at implementation time against the live provider docs, provider APIs, or service-local config.

Provider trust classes, routing lanes, and fallback rules live in [references/provider-router-policy.md](../../../../references/provider-router-policy.md).

If a scaffold run needs project-specific adaptation notes that do not belong in the package, write them into the generated repo instead of mutating this directory.
